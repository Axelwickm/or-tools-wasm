CP-SAT JS API TEST TRACKER

Goal:
Track the upstream API behavior we want this package to guarantee, without trying
to port every internal C++ solver test. The canonical source for CP-SAT API
behavior is the Python API test suite because this package exposes proto-first
solver APIs rather than internal propagator APIs.

Default fixture entrypoint:
tests/fixtures/browser-basic-src/cases/ortools/sat/index.ts

Canonical upstream CP-SAT API sources:
- ortools/sat/python/cp_model_helper_test.py
- ortools/sat/python/cp_model_test.py

Future routing API source:
- ortools/constraint_solver/python/pywraprouting_test.py

Status values:
- implemented: covered by the shared JS fixture corpus.
- todo: API behavior should be covered, but is not implemented yet.
- blocked_missing_api: upstream behavior needs a callback/helper/builder API that
  this package does not expose today.
- not_applicable_no_builder: upstream test only covers Python builder ergonomics,
  operator overloads, or Python-specific data structures.
- future_solver: belongs to a solver/API this package does not expose yet.

double_checked:
- no means the test was categorized or implemented, but has not yet been audited
  line-by-line against upstream.
- yes means the JS fixture has been compared against upstream behavior.
